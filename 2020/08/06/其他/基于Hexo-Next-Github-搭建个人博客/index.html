基于Hexo + Next + Github 搭建个人博客

发表于 2020-08-06 更新于 2020-08-11 分类于 其他

默认已拥有Github账号，且已安装好了git。

Node.js

安装Node.js

下载地址：Node.js 官网

安装过程全程next，最后安装好之后，按Win+R打开命令提示符，输入node -v和npm -v，如果出现版本号，那么就安装成功了。

添加阿里的国内镜像源进行加速，也可以用梯子。

Github

创建github.io仓库

和新建仓库的流程相似，不过在输入项目名字时，后面需要加上github.io后缀，README初始化要勾选。

项目建成后，点击Settings ,拉到最下方Github Pages的位置，点击Choose a theme选择一个主题。选取主题后进行提交。

访问你的博客链接 wzymumon.github.io，就会出现下面的效果：

Hexo

安装Hexo

在合适的地方新建一个文件夹，用来存放自己的博客文件，比如我的博客文件都存放在F:\blog目录下。

在该目录下右键点击Git Bash Here，打开git的控制台窗口，以后我们所有的操作都在git控制台进行。

定位到该目录下，输入npm i hexo-cli -g安装Hexo。会有几个报错，无视它就行。

安装完后输入hexo -v验证是否安装成功。

然后就要初始化我们的网站，输入hexo init初始化文件夹，接着输入npm install安装必备的组件。

这样本地的网站配置也弄好啦，输入hexo g生成静态网页，然后输入hexo s打开本地服务器，然后浏览器打开http://localhost:4000/，就可以看到我们的博客啦，效果如下：

写文章、发布文章

首先在博客根目录下右键打开git bash，安装扩展npm i hexo-deployer-git。

博客上的文章都位于F:\blog\source\_posts的目录下，可以发现该目录下面多了一个文件夹和一个.md文件，一个用来存放你的图片等数据，另一个用来存放你的markdown文档。

备份博客源文件

有时候我们想换一台电脑继续写博客，这时候就可以将博客目录下的所有源文件都上传到github上面。

首先在github博客仓库下新建一个分支hexo，然后git clone到本地，把.git文件夹拿出来，放在博客根目录下。

然后git branch -b hexo切换到hexo分支，然后git add .，然后git commit -m "xxx"，最后git push origin hexo提交就行了。

配置图床

使用typora + PicGo + Github进行搭建图床。

PicGo设置

仓库名 ：即你的仓库名

分支名 ：默认 master

Token ：就是刚刚复制的那一串字符

存储路径： 这个不填，填了的话图片就上传到 git 中 data 这个文件夹

域名 ：https://cdn.jsdelivr.net/gh/github用户名/仓库名@分支

在Typora 的文件目录下,点击偏好设置。
